### Keyword File Format & Configuration

- Keywords are organized with associated UPCs/product IDs for each keyword
- Format shows keywords and which products are applicable to each keyword
- Retailer-specific filtering is required - not all products in keyword list are sold at every retailer
- Team must reference existing file from Susannah to identify which products are available at each retailer
- Example: Allen's may have 20 total products, but only specific ones are sold on Walmart, Metro (12 products), or Loblaws (all 20)
- Each keyword is broken into tabs in the file
- Team can create configuration sheet from the provided format

### SKU Mapping & Coverage

- Relevant SKUs are client products that match the keyword and are sold at the retailer
- Non-relevant SKUs are McCain products that appear in keyword searches but are not in the client's product list
- Non-relevant SKUs are not included in coverage calculations
- Coverage metrics will track performance across relevant SKUs only

### Reporting & Visualization Requirements

- Current format shows one keyword on one retailer with daily tracking over 30 days
- Team can filter by keyword and switch between different keywords (e.g., frozen fries to potato chips or savory snacks)
- Team can switch between retailers (e.g., Allen's to Rougemont) and view different SKU sets
- Request for aggregate view across all keywords for a single retailer showing overall average rank and coverage
- Request for highest-level aggregate showing performance across all retailers in Canada with overall average rank and coverage percentage
- Team suggested building aggregate views similar to content dashboard, showing keywords with metrics like coverage and average ranking at retailer or aggregate level
- Team confirmed aggregation is feasible and not difficult to implement
