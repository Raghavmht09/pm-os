# Implementation Roadmap & Timeline

**Purpose:** Sprint plans, milestone tracking, and implementation schedules for DSIM development.

## What Goes Here

- **Quarterly Roadmaps:** DSIM features and initiatives committed for the quarter
- **Sprint Plans:** Weekly/bi-weekly sprint breakdowns with story mappings
- **Milestone Tracker:** Major delivery dates, PoC launches, beta windows, GA dates
- **Release Calendar:** Versioning, feature freeze dates, deployment schedules
- **Dependency Map:** Cross-team blockers, external partner timelines (Walmart integrations, etc.)

## Naming Convention

- `dsim-roadmap-q[N]-2026.md` — Quarterly roadmap
- `dsim-sprint-[sprint-name]-plan.md` — Individual sprint plans
- `dsim-milestones-2026.md` — Annual milestone tracker
- `dsim-timeline-[feature-name].md` — Feature-specific timeline

## How to Keep It Current

- **Weekly:** Update sprint plans with actual progress, blockers, carry-over items
- **Sprint-end:** Run `/sprint-retro` to capture velocity, carry-over % trends
- **Monthly:** Update milestone tracker against actuals
- **Quarterly:** Plan next quarter roadmap, update quarterly roadmap doc
